## What was refactored

### 1) Code bloat / clarity
- Replaced repeated transaction boilerplate with one `tx()` wrapper.
- Added prepared-statement cache (`stmt()`), removing repeated `db.prepare()` churn in hot paths.
- Consolidated SQL usage into a single `SQL` map and reused statements.
- Reduced duplicated row-fetch/update logic in `putVersion`.

### 2) SSOT behavior mismatches corrected
- **Idempotency replay matching** now includes `object_id` in addition to hash fingerprint to prevent cross-object replay collisions.
- **Session payload ref extraction** enforces required shape keys for session content (`chat_ref`, `active_set`, `inactive_set`, `pinned_set`) and validates Ref contract.
- **Pinned ref validation** now raises explicit pinned-anchor failure (`target_version_id` or `target_object_hash` required).
- **Typed-envelope consistency** remains enforced for canonical fields when both typed and payload values are present.
- **refs_hash determinism** updated to be stable under set-order permutations by hashing normalized ref-path keys for array slots (`/x/0`, `/x/1` -> `/x/*`) while still preserving path class (`/active_set/*` vs `/inactive_set/*`).
- **Canonical JSON serialization** now guarantees JSON-valid canonical strings by JSON round-trip + stable key ordering.

### 3) Performance improvements
- Prepared statement cache for repeated writes/reads.
- Removed extra select for `tx_seq` by using `lastInsertRowid` from version insert.
- Escaped `LIKE` prefix queries and validated `limit` to avoid pathological query shapes.
