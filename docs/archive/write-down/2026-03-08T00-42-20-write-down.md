## Fixed methodology (no flexible branches)

1. **Test-gated delivery**
   - Every SSOT DB invariant is implemented only when a matching automated test exists first or in same change.
   - Acceptance is test outcomes, not code-complete status.

2. **Single concrete runtime choice**
   - SQLite driver: **Node built-in `node:sqlite` with `DatabaseSync`**.
   - No alternate driver path in this wave.

3. **Single storage boundary**
   - Implement and use only `StoragePort` from DB SSOT §9.
   - Runtime/agentic refactor is explicitly out of this wave.

4. **Single schema source**
   - SSOT §3 tables and SSOT §4 indexes implemented exactly as SQL DDL in `src/storage/sqlite-schema.ts`.
   - Schema is applied by `SqliteStorage.migrate()`.

5. **Deterministic hash/reference rules**
   - Canonical JSON serializer: stable key-order stringify.
   - `object_hash` preimage format fixed to SSOT §8.
   - Ref extraction is explicit-only and fixed to session payload fields:
     - `/chat_ref`
     - `/system_prompt_ref`
     - `/active_set/<i>`
     - `/inactive_set/<i>`
     - `/pinned_set/<i>`

6. **Conflict-order rule is mandatory**
   - Idempotency decision checked before optimistic head conflict, per SSOT §5 ordering.
